# 分层记忆体系

框架提供三层记忆架构，按生命周期和粒度分层管理，让 Agent 具备"记住"能力：

```
┌─────────────────────────────────────────────────┐
│ 短期记忆 (agentx_session)                          │
│ 按 conversationId 维度，当前会话的问答历史           │
│ 自动加载 / 自动保存                                │
├─────────────────────────────────────────────────┤
│ 用户画像 (agentx_memory)                           │
│ 按 userId 维度，跨会话的结构化用户信息               │
│ LLM 自动提取 / 智能合并 / 自动去重                  │
├─────────────────────────────────────────────────┤
│ 长期记忆 (agent_semantic_memory)                  │
│ 按 userId 维度，向量化存储问答对和摘要               │
│ 语义检索 / 自动摘要压缩                            │
└─────────────────────────────────────────────────┘
```

## 三层记忆对比

| 层级 | 存储方式 | 维度 | 生命周期 | 典型用途 |
|------|---------|------|---------|---------|
| 短期记忆 | 关系型数据库（JSON） | conversationId | 单次会话 | 当前对话上下文 |
| 用户画像 | 关系型数据库 | userId | 跨会话持久 | 用户偏好、行为指令、客观事实 |
| 长期记忆 | 向量数据库 | userId | 跨会话持久 | 历史知识检索、QA 回忆 |

## 零配置模式

只需传入 DataSource，框架自动创建 `agentx_session` 和 `agentx_memory` 表，管理短期记忆和用户画像：

```java
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .dataSource(dataSource)                           // 自动创建会话表 + 用户画像表
        .build();

RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")
        .userId("user_123")
        .build();

String answer = agent.call("帮我分析这段代码", params);
// 框架自动：加载会话历史 → 注入用户画像 → 对话 → 保存会话 → 提取用户画像
```

### 自动执行流程

```
调用 call/stream
    │
    ├─ 1. 加载短期记忆（按 conversationId 从 agentx_session 加载历史对话）
    ├─ 2. 加载用户画像（按 userId 从 agentx_memory 加载结构化信息）
    │
    ├─ 3. 注入系统提示词（将记忆信息格式化后追加到 system prompt）
    │
    ├─ 4. 执行 Agent 循环（LLM 推理 + 工具调用）
    │
    ├─ 5. 保存短期记忆（将本轮问答保存到 agentx_session）
    ├─ 6. 提取用户画像（LLM 分析对话，提取用户信息，智能合并到 agentx_memory）
    └─ 7. 保存长期记忆（如有 SemanticMemoryStore，将 QA 对向量化存储）
```

## 启用长期记忆

长期记忆需要额外传入 VectorStore 和 EmbeddingModel，框架自动创建 `agent_semantic_memory` 向量表：

```java
// 创建 VectorStore（以 PgVector 为例）
PgVectorStore vectorStore = PgVectorStore.builder(jdbcTemplate, embeddingModel)
        .dimensions(1024)
        .vectorTableName("agent_semantic_memory")
        .initializeSchema(true)
        .build();

// 创建 SemanticMemoryStore
SemanticMemoryStore semanticMemoryStore = new SemanticMemoryStore(vectorStore, embeddingModel);

// 配置 Agent
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .dataSource(dataSource)                           // 短期记忆 + 用户画像
        .semanticMemoryStore(semanticMemoryStore)         // 长期记忆（可选）
        .build();

String answer = agent.call("之前我们讨论过的架构方案是什么", params);
// 框架自动：加载会话历史 → 注入用户画像 → 语义检索相关知识 → 对话 → 保存会话 → 提取画像 → 存入向量
```

### 长期记忆特性

- **QA 对存储**：每轮对话结束后，将问答对向量化存入 `agent_semantic_memory`
- **自动摘要压缩**：当文档数量达到阈值（默认 30 条）时，自动用 LLM 生成摘要，合并相关摘要防止无限增长
- **语义检索**：根据用户问题检索最相关的历史对话知识，注入到上下文中
- **文档类型**：`qa_pair`（原始问答对）和 `summary`（摘要文档）

## 使用不同存储后端

### JDBC（推荐生产环境）

```java
// 使用 Spring Boot 自动注入的 DataSource
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .dataSource(dataSource)         // 自动创建 agentx_session + agentx_memory 表
        .build();
```

### 内存（适合开发测试）

```java
// 不传 DataSource，使用内存存储（重启后丢失）
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .memoryStore(new InMemoryMemoryStore())   // 可选，默认即内存
        .build();
```

### 文件系统

```java
// JSON 文件存储，按 userId 分目录
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .memoryStore(new FileMemoryStore("/data/memories"))  // 指定存储目录
        .build();
```

## 数据库表结构

### agentx_session（短期记忆）

| 列名 | 类型 | 说明 |
|------|------|------|
| id | BIGINT(20) | 主键（雪花算法） |
| conversation_id | VARCHAR(100) | 会话标识 |
| messages | TEXT | 对话消息（JSON 格式） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### agentx_memory（用户画像）

| 列名 | 类型 | 说明 |
|------|------|------|
| id | BIGINT(20) | 主键（雪花算法） |
| user_id | VARCHAR(100) | 用户标识 |
| type | VARCHAR(20) | 记忆类型（PROFILE/PREFERENCE/INSTRUCTION/FACT） |
| content | TEXT | 记忆内容 |
| description | TEXT | 描述 |
| metadata_json | TEXT | 元数据（JSON） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

索引：`idx_agentx_memory_user_id`、`idx_agentx_memory_user_type`

### agent_semantic_memory（长期记忆）

由 VectorStore 管理，存储向量化的 QA 对和摘要。

## 用户画像类型（agentx_memory）

| 类型 | 说明 | 提取示例 |
|------|------|---------|
| PROFILE | 用户画像 | "用户是一名 Java 开发工程师，5 年经验" |
| PREFERENCE | 用户偏好 | "偏好中文回答，喜欢简洁的代码风格" |
| INSTRUCTION | 行为指令 | "不要使用 var 关键字" |
| FACT | 客观事实 | "项目使用 JDK 21，Spring Boot 3.5" |

框架通过 LLM 自动从对话中提取用户画像，与已有信息智能合并：
- **新增**：全新信息直接添加
- **更新**：冲突信息以新为准（如偏好变化）
- **去重**：重复信息自动跳过

## MemoryTool — Agent 主动读写记忆

框架提供 `MemoryTool`，让 Agent 在对话过程中主动读取和保存记忆：

```java
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .dataSource(dataSource)
        .tools(MemoryTool.create(memoryStore))    // 注册记忆工具
        .build();

// Agent 可以主动调用工具：
// - saveMemory：主动保存信息到用户画像
// - getMemory：读取用户的历史画像
// - searchMemory：语义搜索长期记忆
```

## 记忆注入到提示词

记忆信息会格式化后追加到系统提示词中：

```
## 用户信息
- 职业: Java 开发工程师
- 偏好: 中文回答、简洁风格
- 事实: 使用 JDK 21

## 历史对话知识
- 2024-01-15: 用户询问了微服务架构方案，讨论了 Spring Cloud vs Dubbo
- 2024-01-10: 用户偏好 RESTful API 风格
```

LLM 会基于这些记忆信息提供更个性化的回答。

## 最佳实践

1. **务必传入 conversationId 和 userId**：这是记忆体系工作的前提
2. **生产环境使用 JDBC**：内存存储重启后丢失，不适合生产
3. **长期记忆需要 EmbeddingModel**：选择与你的 LLM 兼容的 Embedding 模型
4. **记忆提取消耗额外 Token**：框架会在对话后额外调用 LLM 提取记忆，注意成本
5. **使用 MemoryTool 增强 Agent 记忆能力**：Agent 可以主动保存和检索记忆
