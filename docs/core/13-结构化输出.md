# 结构化输出

框架支持在非流式调用（`call()` / `callForResult()`）中，让 LLM 按指定的 JSON Schema 输出结构化数据。调用方通过 `RunnableParams.outputType` 传入目标类型，框架自动注入格式指令到 system prompt，并在返回前修复 LLM 输出为合法 JSON。

> 此功能仅适用于非流式调用。流式输出（`stream()`）以增量文本片段实时推送，JSON 片段不完整且无法修复，因此不支持结构化输出。

## 定义 POJO

结构化输出的目标类型是一个标准 Java POJO，使用 Jackson 注解映射 JSON 字段。推荐使用 Lombok `@Data` 简化代码：

```java
import com.fasterxml.jackson.annotation.JsonProperty;
import com.fasterxml.jackson.annotation.JsonPropertyDescription;
import lombok.Data;

@Data
public class BookInfo {

    @JsonProperty(value = "title")
    @JsonPropertyDescription("书籍名称")
    private String title;

    @JsonProperty(value = "author")
    @JsonPropertyDescription("书籍作者")
    private String author;

    @JsonProperty(value = "year")
    @JsonPropertyDescription("出版年份，四位数整数，如 2024")
    private int year;

    @JsonProperty(value = "summary")
    @JsonPropertyDescription("书籍简要介绍，100字以内")
    private String summary;
}
```

### 注解说明

| 注解 | 用途 | 示例 |
|------|------|------|
| `@JsonProperty(value = "name")` | 指定 JSON 中的 key 名称 | `@JsonProperty(value = "title")` |
| `@JsonPropertyDescription("描述")` | 为字段生成描述，写入 JSON Schema 帮助 LLM 理解 | `@JsonPropertyDescription("出版年份，四位数整数，如 2024")` |
| `@JsonIgnore` | 忽略该字段，不纳入 JSON Schema | `@JsonIgnore private String internalId;` |

### POJO 编写要点

1. **必须提供无参构造器**：Jackson 反序列化需要，使用 Lombok `@Data` 时会自动生成
2. **`@JsonPropertyDescription` 是关键**：描述越精确，LLM 输出越准确。建议包含：
   - 字段含义说明
   - 允许的取值范围（如"仅允许取值为：A、B、C"）
   - 输出格式要求（如"100字以内"、"四位数整数"）
3. **不需要手写 getter/setter**：使用 Lombok `@Data` 即可

## 使用方式

### 单对象输出

```java
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .maxRounds(3)
        .build();

RunnableParams params = RunnableParams.builder()
        .outputType(BookInfo.class)        // 直接传 Class
        .build();

String json = agent.call("推荐一本 Java 编程经典书籍", params);

// json 是合法 JSON，自行反序列化
BookInfo book = new ObjectMapper().readValue(json, BookInfo.class);
```

### List 集合输出

```java
RunnableParams params = RunnableParams.builder()
        .outputType(OutputType.listOf(BookInfo.class))   // 泛型集合
        .build();

String json = agent.call("推荐 3 本编程经典书籍", params);

// 反序列化为 List<BookInfo>
List<BookInfo> books = new ObjectMapper().readValue(json,
        new TypeReference<List<BookInfo>>() {});
```

> Java 泛型存在类型擦除，无法直接写 `List<BookInfo>.class`。使用 `OutputType.listOf()` 传入元素类型即可，框架内部自动处理。

### callForResult 结构化输出

```java
RunnableParams params = RunnableParams.builder()
        .outputType(BookInfo.class)
        .build();

AgentResult result = agent.callForResult("推荐一本 Spring 经典书籍", params);

if (result instanceof AgentResult.Completed c) {
    // c.answer() 保证是合法 JSON
    BookInfo book = new ObjectMapper().readValue(c.answer(), BookInfo.class);
}
```

### OutputType 工厂方法

| 方法 | 说明 | 生成的 JSON 结构 |
|------|------|-----------------|
| `OutputType.of(BookInfo.class)` | 单对象 | `{}` |
| `OutputType.listOf(BookInfo.class)` | List 集合 | `[{}, {}]` |

`RunnableParams.builder().outputType(BookInfo.class)` 是 `outputType(OutputType.of(BookInfo.class))` 的简写。

## 工作原理

框架在两个环节处理结构化输出：

1. **请求前**：将 `OutputType` 转为 JSON Schema 格式指令，追加到 system prompt，指导 LLM 按格式输出
2. **返回前**：通过 `JsonRepairUtil.fixJson()` 自动修复 LLM 输出中的常见问题（Markdown 代码块、中文引号、尾部逗号、缺失引号等），确保返回合法 JSON

不传 `outputType` 时 `call()` 行为完全不变，正常返回自由文本。

## 最佳实践

1. **POJO 必须有无参构造器**：Jackson 反序列化需要
2. **用 `@JsonPropertyDescription` 描述字段**：帮助 LLM 理解每个字段的含义，提高输出准确率
3. **配合 instructions 效果更好**：通过 `instructions` 约束 LLM 的回答风格，能进一步提高结构化输出的质量
4. **outputType 是 per-call 参数**：放在 `RunnableParams` 中，可复用同一个 Agent 实例、不同调用传不同类型
