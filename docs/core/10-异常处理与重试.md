# 异常处理与重试

框架内置透明的 LLM 调用重试机制，当 LLM 调用失败（网络异常、服务端错误、空响应等）时自动重试。重试行为由 `maxRetries` 控制，默认 3 次，设为 0 禁用。

## 为什么需要重试

LLM 调用可能因为多种原因失败：
- **网络抖动**：临时性连接超时、DNS 解析失败
- **服务端错误**：API 服务过载、限流（429/500/502/503）
- **空响应**：模型返回空结果

框架在 Agent 执行循环内部透明处理这些异常，调用方无需手动重试。

## 配置重试

```java
// 默认 maxRetries=3（推荐）
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .build();

// 高可用场景，增加重试次数
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .maxRetries(5)
        .build();

// 禁用重试（需要自行处理所有异常）
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .maxRetries(0)
        .build();
```

## 四种 API 的异常行为

不同 API 方法对异常的处理方式不同，选择合适的 API 可以简化错误处理：

| API | 返回类型 | 异常行为 | 适用场景 |
|-----|---------|---------|---------|
| `call()` | `String` | 重试耗尽后抛出 `AgentException` | 简单脚本，愿意 try-catch |
| `callForResult()` | `AgentResult` | 返回 `AgentResult.Failed`，不抛异常 | **推荐**：模式匹配处理 |
| `stream()` | `Flux<String>` | Error 事件被过滤，流静默结束 | 只需文本，不关心错误 |
| `streamForResult()` | `Flux<AgentStreamEvent>` | 发出 `Error` + `Complete` 事件，流正常结束 | **推荐**：可感知重试状态 |

## 推荐用法

### 同步调用 — callForResult（推荐）

不抛异常，通过模式匹配处理所有结果：

```java
AgentResult result = agent.callForResult("帮我分析这段代码", params);

switch (result) {
    case AgentResult.Completed c -> {
        System.out.println("回答: " + c.answer());
        c.stageOutputs().forEach((stage, data) ->
                System.out.println("[" + stage + "] " + data));
    }
    case AgentResult.Paused p -> {
        // 处理暂停（详见 Human-in-the-Loop）
    }
    case AgentResult.Failed f -> {
        System.out.println("执行失败: [" + f.code() + "] " + f.error());
    }
}
```

### 同步调用 — call（便捷方法）

需要 try-catch，适合快速脚本：

```java
try {
    String answer = agent.call("帮我分析这段代码", params);
    System.out.println("回答: " + answer);
} catch (AgentException e) {
    System.out.println("执行失败: [" + e.getCode() + "] " + e.getMessage());
}
```

### 流式调用 — streamForResult（推荐）

通过事件流感知重试状态：

```java
agent.streamForResult("帮我分析这段代码", params)
        .doOnNext(event -> {
            switch (event) {
                case AgentStreamEvent.Text t -> System.out.print(t.content());
                case AgentStreamEvent.Error e ->
                        System.out.println("[" + e.code().code() + "] " + e.message() + "\n  " + e.detail());
                case AgentStreamEvent.Complete c -> System.out.println("\n完成");
                default -> {}
            }
        })
        .blockLast();
```

### 流式调用 — stream（便捷方法）

自动过滤 Error 事件，只输出文本。如果 LLM 调用失败，流会静默结束（无文本输出）：

```java
agent.stream("帮我分析这段代码", params)
        .doOnNext(chunk -> System.out.print(chunk))
        .blockLast();
```

## AgentErrorCode 错误码

框架通过 `AgentException` 和 `AgentErrorCode` 统一异常处理：

| 错误码 | 枚举值 | 说明 | 触发场景 |
|--------|--------|------|---------|
| `E1001` | `LLM_CALL_FAILED` | LLM 调用失败 | 网络异常、API Key 错误、服务端错误，重试耗尽后触发，message 透传 LLM 原始错误 |
| `E1002` | `LLM_EMPTY_RESPONSE` | LLM 返回空响应 | LLM 返回 null 或空结果 |
| `E2001` | `CONCURRENT_EXECUTION` | 会话并发冲突 | 同一 conversationId 的请求正在执行中 |

### 错误码结构

- `E1xxx`：LLM 相关错误（调用失败、空响应）
- `E2xxx`：执行控制错误（并发冲突）

### 错误处理示例

```java
try {
    String answer = agent.call("hello");
} catch (AgentException e) {
    switch (e.getCode()) {
        case LLM_CALL_FAILED -> {
            // LLM 调用失败，e.getMessage() 为 LLM 原始错误
            // 可能原因：API Key 错误、网络异常、服务端错误
        }
        case LLM_EMPTY_RESPONSE -> {
            // LLM 返回空响应
            // 可能原因：模型过载、请求参数问题
        }
        case CONCURRENT_EXECUTION -> {
            // 并发冲突
            // 解决方案：等待当前任务完成或使用 stopStream 取消
        }
    }
}
```

## 流式重试通知

流式模式下（`streamForResult`），每次重试会发出 `Error` 事件通知调用方：

### 重试中

```
[Text] 部分文本...                                              ← LLM 输出中断
[Error:E1001] LLM 调用失败，正在重试 (1/3)                      ← 重试通知
  invalid_api_key                                              ← LLM 原始错误透传
[Text] 完整的文本输出...                                         ← 重试成功后的新输出
[Complete]
```

### 重试耗尽

```
[Error:E1001] LLM 调用失败（已重试 3 次）
  Failed to resolve 'api.minimaxi.com' after 4 queries         ← LLM 原始错误透传
[Complete]                                                      ← 流正常结束
```

### 在 UI 中展示重试

```java
agent.streamForResult("帮我分析代码", params)
        .doOnNext(event -> {
            if (event instanceof AgentStreamEvent.Error e) {
                // 展示重试提示
                if (e.message().contains("重试")) {
                    ui.showRetryNotification(e.message());
                } else {
                    ui.showError(e.code().code(), e.detail());
                }
            }
        })
        .blockLast();
```

## AgentException 结构

```java
public class AgentException extends RuntimeException {
    private final AgentErrorCode code;   // 错误码

    public AgentErrorCode getCode() { return code; }
    public String getMessage() { ... }   // 错误详情
}
```

## 最佳实践

1. **生产环境用高级 API**：`callForResult` / `streamForResult` 不抛异常，不会中断业务流程
2. **UI 展示重试状态**：通过 `streamForResult` 的 Error 事件实时展示重试进度
3. **错误日志记录 detail**：Error 事件的 `detail` 字段包含 LLM 原始错误，有助于排查
4. **合理设置重试次数**：3 次适合大多数场景，关键业务可设 5 次
5. **区分错误类型**：`LLM_CALL_FAILED` 多为临时问题，`CONCURRENT_EXECUTION` 需要业务处理
