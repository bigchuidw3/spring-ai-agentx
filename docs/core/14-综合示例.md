# 综合示例：Skills + HITL + 分阶段输出

下面以一个完整的旅游规划场景为例，展示 Skills、Human-in-the-Loop 和分阶段输出三个能力如何协同工作。

> 本示例综合使用了以下功能：
> - [Skills 技能体系](08-Skills技能体系.md) — 按需加载技能
> - [Human-in-the-Loop](07-Human-in-the-Loop.md) — 用户问答与操作审批
> - [分阶段输出](06-分阶段输出.md) — 工具摘要、引用、推荐
> - [同步与流式输出](01-同步与流式输出.md) — streamForResult + resumeStream
> - [Think 标签解析](09-Think标签解析.md) — 支持思考模型

## 场景描述

用户请求"帮我规划一个旅游行程，并输出到当前目录下的 markdown"。Agent 需要：
1. 通过 `ask_user` 多次追问用户偏好（目的地、预算等）
2. 使用 `write_file` 将行程写入文件（需人工确认）
3. 每次工具执行后输出工具摘要（`AFTER_TOOL_END`）
4. 最终输出引用和推荐问题（`BEFORE_COMPLETE`）

## Agent 配置

```java
// 注册工具：文件系统 + Skills
ToolCallback[] allTools = ReactAgent.mergeTools(
        FileSystemTools.create(),
        new ToolCallback[]{SkillsTool.builder().addSkillsDirectory(skillsDir).build()}
);

ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .tools(allTools)
        .askUser(true)                              // 自动注册 AskUserTool + PauseAdvisor("ask_user")
        .thinkTagEnabled(true)                       // 启用 <think/> 标签解析（支持 DeepSeek/Qwen3 等思考模型）
        .stageOutputProviders(
                new ToolSummaryProvider(),            // AFTER_TOOL_END — 每次工具执行后输出摘要
                new ReferenceProvider(),              // BEFORE_COMPLETE — 提取引用
                new RecommendProvider()               // BEFORE_COMPLETE — 推荐问题
        )
        .advisors(PauseAdvisor.builder()                     // 拦截文件操作（审批工具）
                .approvalTools("write_file", "read_file")
                .build())
        .maxRounds(50)
        .build();
```

### 配置解读

| 配置 | 说明 |
|------|------|
| `askUser(true)` | 自动注册 AskUserTool + PauseAdvisor，ask_user 工具被标记为用户输入工具 |
| `thinkTagEnabled(true)` | 支持 DeepSeek/Qwen3.5 等思考模型的 `<think/>` 标签 |
| `stageOutputProviders(...)` | 注册 3 个分阶段输出 Provider |
| `advisors(PauseAdvisor...)` | 额外拦截 write_file、read_file，标记为操作审批工具 |
| `maxRounds(50)` | HITL 场景需要更多轮次 |

## 事件处理循环

```java
Scanner scanner = new Scanner(System.in);
System.out.print("请输入你的问题: ");
String query = scanner.nextLine();

// 首次流式调用
PauseState pauseState = collectStreamEvents(agent.streamForResult(query, RunnableParams.empty()));

// 循环处理暂停
while (pauseState != null) {
    System.out.println("--- Agent 暂停 ---");
    Map<String, String> answers = new LinkedHashMap<>();

    for (PendingToolCall ptc : pauseState.getPendingToolCalls()) {
        System.out.println("[" + ptc.name() + "] " + ptc.arguments());

        if ("ask_user".equals(ptc.name())) {
            System.out.print("你的回答: ");
        } else {
            System.out.print("确认执行 " + ptc.name() + "（ok/拒绝）: ");
        }
        answers.put(ptc.id(), scanner.nextLine());
    }

    // 恢复执行
    pauseState = collectStreamEvents(agent.resumeStream(pauseState, answers));
}
```

### 事件收集辅助方法

```java
private static PauseState collectStreamEvents(Flux<AgentStreamEvent> flux) {
    PauseState[] holder = new PauseState[1];
    flux.doOnNext(event -> {
        if (event instanceof AgentStreamEvent.Paused p) {
            holder[0] = p.state();           // 捕获暂停状态
        } else {
            printEvent(event);                // 实时打印其他事件
        }
    }).blockLast();
    return holder[0];
}

private static void printEvent(AgentStreamEvent event) {
    switch (event) {
        case AgentStreamEvent.AgentStart s -> System.out.println("[AgentStart]");
        case AgentStreamEvent.Thinking t -> System.out.println("[Thinking] " + t.content());
        case AgentStreamEvent.Text t -> System.out.print(t.content());
        case AgentStreamEvent.ToolStart ts -> System.out.println("[ToolStart] " + ts.toolName());
        case AgentStreamEvent.ToolEnd te -> System.out.println("[ToolEnd] " + te.toolName());
        case AgentStreamEvent.StageOutput so -> System.out.println("[StageOutput:" + so.stage() + "] " + so.data());
        case AgentStreamEvent.Error e -> System.out.println("[Error] " + e.message());
        case AgentStreamEvent.Complete c -> System.out.println("[Complete]");
        default -> {}
    }
}
```

## 完整事件流时序

```
用户：帮我规划一个旅游行程，并输出到当前目录下的 markdown

[AgentStart]
[StageOutput:welcome] {message=Agent 开始处理您的请求}    ← AFTER_START

[Thinking] 用户想要规划旅游行程...                         ← think 模型思考过程
[Text]

[ToolStart] ask_user                                      ← 用户输入工具
[ToolEnd]   ask_user
[StageOutput:tool_summary] {tool=ask_user, status=completed}  ← AFTER_TOOL_END
[Paused]

--- Agent 暂停 ---
[ask_user] {"question": "请告诉我您的旅行偏好", "options": ["海岛度假(推荐)", "城市观光", "户外探险"]}

                        ← ask_user 是用户输入工具，回答直接作为工具结果，不执行

...（多轮 ask_user 交互）...

[Thinking] 用户确认了所有细节，现在生成行程...
[Text]

[ToolStart] write_file                                    ← 操作工具
[Paused]

--- Agent 暂停 ---
[write_file] {"filePath": "hainan_trip.md", "content": "# 海南5天自由行..."}
确认执行 write_file（ok/拒绝）: ok

--- 恢复 Agent 执行 ---
                        ← write_file 是操作工具，用户确认后实际执行写文件

[ToolEnd]   write_file                                    ← 确认后框架实际执行
[StageOutput:tool_summary] {tool=write_file, status=completed}  ← AFTER_TOOL_END

[Thinking] 文件已成功写入...
[Text] 行程已生成到 hainan_trip.md...

[StageOutput:reference] [{tool=write_file, summary=...}]    ← BEFORE_COMPLETE
[StageOutput:recommend] [继续深入了解..., 相关主题推荐...]    ← BEFORE_COMPLETE
[Complete]
```

## 关键交互点解读

### 1. ask_user 暂停

```
[ToolStart] ask_user → [Paused]
```

ask_user 是**用户输入工具**，PauseAdvisor 拦截后暂停。用户回答直接作为工具结果，框架不执行 ask_user 方法体。

### 2. write_file 暂停

```
[ToolStart] write_file → [Paused]
```

write_file 是**操作工具**，PauseAdvisor 拦截后暂停。用户确认后框架实际执行写文件操作。

### 3. 分阶段输出时机

- `AFTER_TOOL_END`：每次工具执行后实时推送（工具摘要）
- `BEFORE_COMPLETE`：最终答案后推送（引用、推荐）

### 4. Thinking 事件

启用 `thinkTagEnabled` 后，思考模型的 `<think/>` 标签内容作为独立 `Thinking` 事件推送，不在 `Text` 事件中出现。

## 适配 Web 场景

将上述 CLI 示例适配为 Web 服务（SSE + REST API）：

```java
@RestController
@RequestMapping("/api/agent")
public class AgentController {

    private final ReactAgent agent;

    // SSE 流式调用
    @GetMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<AgentStreamEvent> stream(@RequestParam String query,
                                          @RequestParam String conversationId) {
        RunnableParams params = RunnableParams.builder()
                .conversationId(conversationId)
                .build();
        return agent.streamForResult(query, params);
    }

    // 恢复暂停的执行
    @PostMapping("/resume")
    public Flux<AgentStreamEvent> resume(@RequestBody ResumeRequest request) {
        return agent.resumeStream(request.pauseState(), request.answers());
    }
}
```

前端通过 SSE 接收事件流，遇到 `Paused` 事件时展示交互界面，用户操作后调用 `/resume` 接口恢复执行。
