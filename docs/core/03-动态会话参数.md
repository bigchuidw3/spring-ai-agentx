# 动态会话参数

通过 `RunnableParams` 传递会话级别的参数，控制会话标识、用户标识和运行时参数注入。`RunnableParams` 在每次 Agent 调用时传入，影响会话记忆加载、任务管理和工具行为。

## 参数字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversationId` | `String` | 会话标识，用于 ChatMemory 会话记忆和 TaskManager 任务管理 |
| `userId` | `String` | 用户标识，用于长期记忆（用户画像）和语义记忆的维度 |
| `customParams` | `Map<String, Object>` | 自定义参数，注入到系统提示词中供 LLM 可见 |
| `toolParams` | `Map<String, Object>` | 工具参数，运行时替换工具调用中的参数值 |

## 基本用法

### 会话和用户标识

```java
RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")       // 会话标识
        .userId("user_123")               // 用户标识
        .build();

agent.call("帮我分析这段代码", params);
```

- `conversationId`：相同 ID 的多次调用共享会话记忆（短期记忆），框架自动加载历史对话
- `userId`：相同 ID 跨会话共享长期记忆（用户画像、语义记忆）

### 提示词参数注入

通过 `addParam` 注入参数到系统提示词，LLM 可以看到这些参数值：

```java
RunnableParams params = RunnableParams.builder()
        .addParam("language", "zh-CN")       // 注入: language: zh-CN
        .addParam("format", "markdown")      // 注入: format: markdown
        .build();

agent.call("写一段自我介绍", params);
```

系统提示词中会追加：

```
language: zh-CN
format: markdown
```

LLM 会根据这些参数调整输出。

## addParam 和 addToolParam 配合

当参数值较长或敏感（如文件路径、API Key），LLM 容易传错时，用 `addParam` + `addToolParam` 配合隐藏真实值：

| 场景 | 提示词注入 | 工具执行时 |
|------|-----------|-----------|
| 只有 `addParam("language", "zh-CN")` | `language: zh-CN`（LLM 可见真实值） | 不替换 |
| `addParam("path", "/data")` + `addToolParam("path", "/data")` | `path: default`（隐藏真实值） | `"default"` → `"/data"` |

**工作原理**：
1. `addParam` 声明参数，如果有对应的 `addToolParam`，提示词中只显示 `default`
2. `addToolParam` 标记需要运行时替换的参数
3. 工具执行时，框架自动将工具参数中的 `"default"` 替换为真实值

```java
RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")
        .addParam("path", "/data/files")          // 提示词中显示: path: default
        .addToolParam("path", "/data/files")      // 工具执行时替换 default → /data/files
        .build();

agent.call("帮我搜索文件", params);
```

> `addToolParam` 必须配合 `addParam` 使用。单独使用 `addToolParam` 无效。

## 完整示例

```java
RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")               // 会话记忆维度
        .userId("user_123")                       // 长期记忆维度
        .addParam("language", "zh-CN")            // LLM 可见的偏好参数
        .addParam("output_format", "json")        // LLM 可见的格式参数
        .addParam("db_connection", "jdbc:postgresql://...")   // 敏感参数，配合 addToolParam 隐藏
        .addToolParam("db_connection", "jdbc:postgresql://...") // 运行时替换
        .build();

agent.call("查询用户列表", params);
```

## 内部实现

- `customParams` 中的参数在构建系统提示词时追加到末尾，LLM 可以感知
- `toolParams` 中的参数在工具执行拦截时，替换工具调用参数中的 `"default"` 值
- 配合 `ToolContext`，工具可以从上下文中获取 `userId` 等信息
