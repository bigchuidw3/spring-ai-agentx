# 分阶段输出（Stage Output）

框架提供 `StageOutputProvider` SPI，允许调用方在 Agent 执行生命周期的关键节点注入自定义输出。适用于引用提取、推荐问题、工具摘要、欢迎语等场景。

与 [同步与流式输出](01-同步与流式输出.md) 的高级 API（`callForResult` / `streamForResult`）配合使用。

## 核心概念

```
Agent 执行生命周期：

AgentStart
  │
  ▼
  StageOutput (AFTER_START)    ← 欢迎语、初始化提示
  │
  ▼
┌─── Loop（多轮循环） ─────────────────────┐
│  Thinking / Text                           │
│  ToolStart → ToolEnd                       │
│  StageOutput (AFTER_TOOL_END)              │  ← 工具摘要、进度追踪
│  Paused → Complete                         │
└──────────────────────────────────────────┘
  │
  ▼
  StageOutput (BEFORE_COMPLETE)  ← 引用链接、推荐问题、结果摘要
  Error / Complete
```

## StageTiming — 三种钩子时机

| 钩子时机 | 触发点 | StageContext 可用信息 | 典型用途 |
|----------|--------|---------------------|---------|
| `AFTER_START` | AgentStart 之后，第一轮 LLM 调用之前 | `query`, `params` | 欢迎语、初始化提示、参数校验 |
| `AFTER_TOOL_END` | 每次工具执行完成后 | `query`, `params`, `toolRecords`（含最新记录） | 工具摘要、实时卡片、进度追踪 |
| `BEFORE_COMPLETE` | 最终答案确定后，Complete 之前 | `query`, `answer`, `params`, `toolRecords`（全部记录） | 引用链接、推荐问题、结果摘要 |

## StageContext — 钩子上下文

每次调用 `StageOutputProvider.produce()` 时传入的上下文信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| `query` | `String` | 原始用户问题 |
| `answer` | `String` | 最终答案（仅 `BEFORE_COMPLETE` 时有值，其他时机为 null） |
| `params` | `RunnableParams` | 调用参数（conversationId、userId 等） |
| `toolRecords` | `List<ToolRecord>` | 所有工具执行记录（`AFTER_TOOL_END` 时包含到当前为止的记录） |
| `lastToolRecord()` | `ToolRecord` | 快捷方法，获取最后一个工具记录（可能为 null） |

## 实现 StageOutputProvider

### 接口定义

```java
public interface StageOutputProvider {
    String name();                          // 阶段标识，用于 StageOutput 事件中区分来源
    StageTiming timing();                   // 钩子时机
    Object produce(StageContext context);   // 生成输出，返回 null 表示本阶段无输出
}
```

### 示例：引用提取（BEFORE_COMPLETE）

在最终答案后提取工具调用中的引用信息：

```java
public class ReferenceProvider implements StageOutputProvider {
    @Override
    public String name() {
        return "reference";  // StageOutput 事件的 stage 字段值
    }

    @Override
    public StageTiming timing() {
        return StageTiming.BEFORE_COMPLETE;
    }

    @Override
    public Object produce(StageContext context) {
        if (context.toolRecords().isEmpty()) {
            return null;  // 无工具调用，不输出
        }
        return context.toolRecords().stream()
                .map(r -> Map.of("tool", r.toolName(), "summary", r.result()))
                .toList();
    }
}
```

### 示例：推荐问题（BEFORE_COMPLETE）

基于用户提问生成推荐：

```java
public class RecommendProvider implements StageOutputProvider {
    @Override
    public String name() { return "recommend"; }

    @Override
    public StageTiming timing() { return StageTiming.BEFORE_COMPLETE; }

    @Override
    public Object produce(StageContext context) {
        String query = context.query();
        if (query == null || query.isEmpty()) return null;
        return List.of("继续深入了解：" + query, "相关主题推荐");
    }
}
```

### 示例：工具摘要（AFTER_TOOL_END）

每次工具执行后实时输出摘要：

```java
public class ToolSummaryProvider implements StageOutputProvider {
    @Override
    public String name() { return "tool_summary"; }

    @Override
    public StageTiming timing() { return StageTiming.AFTER_TOOL_END; }

    @Override
    public Object produce(StageContext context) {
        ToolRecord last = context.lastToolRecord();
        if (last == null) return null;
        return Map.of(
            "tool", last.toolName(),
            "status", "completed",
            "resultLength", last.result().length()
        );
    }
}
```

### 示例：欢迎语（AFTER_START）

Agent 启动后发送欢迎消息：

```java
public class WelcomeProvider implements StageOutputProvider {
    @Override
    public String name() { return "welcome"; }

    @Override
    public StageTiming timing() { return StageTiming.AFTER_START; }

    @Override
    public Object produce(StageContext context) {
        return Map.of(
            "message", "Agent 开始处理您的请求",
            "query", context.query()
        );
    }
}
```

## 注册 Provider

```java
ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .tools(FileSystemTools.create())
        .stageOutputProviders(
                new WelcomeProvider(),       // AFTER_START
                new ToolSummaryProvider(),   // AFTER_TOOL_END
                new ReferenceProvider(),     // BEFORE_COMPLETE
                new RecommendProvider()      // BEFORE_COMPLETE
        )
        .build();
```

多个 Provider 可以注册在同一钩子时机，它们会按注册顺序依次调用。

## 消费 StageOutput 事件

### 流式模式（推荐）

通过 `streamForResult` 获取完整事件流，`StageOutput` 事件会与其他事件一起推送：

```java
agent.streamForResult("读取 pom.xml 的内容", RunnableParams.empty())
        .doOnNext(event -> {
            switch (event) {
                case AgentStreamEvent.Text t -> System.out.print(t.content());
                case AgentStreamEvent.StageOutput so ->
                        System.out.println("[" + so.stage() + "] " + so.data());
                case AgentStreamEvent.Complete c -> System.out.println("完成");
                default -> {}
            }
        })
        .blockLast();
```

`StageOutput` 事件的 `stage` 字段对应 Provider 的 `name()`，`data` 字段对应 `produce()` 的返回值。

### 同步模式

通过 `callForResult` 获取 `AgentResult.Completed`，其中包含 `stageOutputs` Map：

```java
AgentResult result = agent.callForResult("读取 pom.xml 的内容", RunnableParams.empty());

if (result instanceof AgentResult.Completed c) {
    System.out.println("回答: " + c.answer());
    c.stageOutputs().forEach((stage, data) ->
            System.out.println("[" + stage + "] " + data));
}
```

注意：同步模式下，同一钩子时机如果有多个 Provider，后注册的会覆盖前面的（Map 去重）。如需保留所有输出，建议使用流式模式。

## 完整事件类型参考

通过 `streamForResult` 获取的 `AgentStreamEvent` 流包含以下事件类型：

| 事件类型 | 字段 | 说明 |
|---------|------|------|
| `AgentStart` | 无 | Agent 开始执行，流中第一个事件 |
| `Thinking` | `content` | LLM 思考过程（需配置 `thinkingMode`，支持 `REASONING_CONTENT`） |
| `Text` | `content` | LLM 正常文本输出，逐 chunk 推送 |
| `ToolStart` | `toolName`, `toolCallId`, `arguments` | 工具即将执行 |
| `ToolEnd` | `toolName`, `toolCallId`, `result` | 工具执行完成 |
| `Paused` | `state`（PauseState） | 执行暂停 |
| `StageOutput` | `stage`, `data` | 自定义阶段输出 |
| `Error` | `code`, `message`, `detail` | LLM 调用失败通知 |
| `Complete` | 无 | Agent 执行完成 |

## 内部实现

- `StageOutputManager` 按 `StageTiming` 分组管理 Provider
- 同步路径：输出收集到 `Map<String, Object>`，通过 `AgentResult.Completed.stageOutputs()` 返回
- 流式路径：通过 `Sinks.Many<AgentStreamEvent>` 实时推送 `StageOutput` 事件
- `produce()` 返回 null 时不产生输出，不会发送 `StageOutput` 事件

### 思考模型适配

配置 `thinkingMode()` 后，框架自动解析模型的思考内容：
- `ThinkingMode.THINK_TAG`：解析 `<think/>` 标签，标签内内容作为 `AgentStreamEvent.Thinking` 事件
- `ThinkingMode.REASONING_CONTENT`：解析 `reasoning_content` 字段，直接作为 `AgentStreamEvent.Thinking` 事件
- 支持 DeepSeek、Qwen3.6-plus、MiniMax 等思考模型

## 最佳实践

1. **Provider 的 name() 要唯一**：用于区分不同 Provider 的输出
2. **produce() 要快速**：不要在 produce() 中执行耗时操作，会阻塞 Agent 执行
3. **善用 null 返回**：不需要输出时返回 null，避免无意义的空输出
4. **AFTER_TOOL_END 注意高频**：工具调用多时会频繁触发，避免产生过多数据
5. **同步模式用 Map 注意覆盖**：同一 timing 的多个 Provider，后注册的会覆盖前面的 name
