# 同步与流式输出

框架提供两套 API × 两种调用模式，覆盖从简单到复杂的全部场景：

| API | 同步返回 | 流式返回 | 适用场景 |
|-----|---------|---------|---------|
| **基础 API** | `call()` → `String` | `stream()` → `Flux<String>` | 只需要最终文本结果 |
| **高级 API** | `callForResult()` → `AgentResult` | `streamForResult()` → `Flux<AgentStreamEvent>` | 需要状态追踪、事件感知 |

## 如何选择

- **只需要答案文本** → 基础 API（`call` / `stream`）
- **需要感知执行过程**（工具调用、重试状态等） → 高级 API（`streamForResult`）
- **需要处理失败场景** → 高级 API（`callForResult` 返回 `Failed`，不抛异常）
- **Web 应用实时推送** → `streamForResult` + SSE / WebSocket

## 基础 API

### 同步调用

阻塞等待完整结果，适用于批处理、测试等不需要实时输出的场景：

```java
String answer = agent.call("帮我分析一下 Java 和 Go 的优劣势");
```

### 流式输出

基于 Reactor Flux，逐 token 实时返回，适用于 Chat UI、实时展示等场景：

```java
agent.stream("用三句话介绍 Spring AI")
        .doOnNext(chunk -> System.out.print(chunk))
        .blockLast();
```

> `stream()` 只返回文本，`Error` 事件被自动过滤。如果 LLM 调用失败，流会静默结束。如需感知错误，请使用 `streamForResult`。

## 高级 API

### AgentResult 类型

`callForResult()` 返回 `AgentResult` 密封接口，有三个子类型：

| 类型 | 说明 | 字段 |
|------|------|------|
| `AgentResult.Completed` | 执行完成 | `answer`（回答文本）、`stageOutputs`（分阶段输出 Map） |
| `AgentResult.Paused` | 执行暂停（等待人工介入） | `state`（PauseState，包含完整暂停上下文） |
| `AgentResult.Failed` | 执行失败 | `error`（错误信息）、`code`（错误码） |

```java
AgentResult result = agent.callForResult("帮我分析代码", params);

switch (result) {
    case AgentResult.Completed c -> System.out.println("回答: " + c.answer());
    case AgentResult.Paused p -> System.out.println("暂停: " + p.state().getPendingToolCalls());
    case AgentResult.Failed f -> System.out.println("失败: [" + f.code() + "] " + f.error());
}
```

> `Paused` 的暂停恢复机制详见 [07-Human-in-the-Loop](07-Human-in-the-Loop.md)。

### AgentStreamEvent 类型

`streamForResult()` 返回 `Flux<AgentStreamEvent>`，包含以下事件类型：

| 事件 | 字段 | 说明 |
|------|------|------|
| `AgentStart` | 无 | Agent 开始执行 |
| `Thinking` | `content` | LLM 思考过程（需启用 `thinkTagEnabled`） |
| `Text` | `content` | LLM 正常文本输出 |
| `ToolStart` | `toolName`, `toolCallId`, `arguments` | 工具即将执行 |
| `ToolEnd` | `toolName`, `toolCallId`, `result` | 工具执行完成 |
| `Paused` | `state` | 执行暂停 |
| `StageOutput` | `stage`, `data` | 自定义阶段输出 |
| `Error` | `code`, `message`, `detail` | LLM 调用失败通知 |
| `Complete` | 无 | 执行完成 |

```java
agent.streamForResult("帮我分析代码", params)
        .doOnNext(event -> {
            switch (event) {
                case AgentStreamEvent.Text t -> System.out.print(t.content());
                case AgentStreamEvent.ToolStart ts -> System.out.println("[工具] " + ts.toolName());
                case AgentStreamEvent.ToolEnd te -> System.out.println("[完成] " + te.toolName());
                case AgentStreamEvent.Complete c -> System.out.println("\n[结束]");
                default -> {}
            }
        })
        .blockLast();
```

> 各事件的详细说明分散在对应的功能文档中：`Paused` 详见 [07-Human-in-the-Loop](07-Human-in-the-Loop.md)，`StageOutput` 详见 [06-分阶段输出](06-分阶段输出.md)，`Error` 详见 [10-异常处理与重试](10-异常处理与重试.md)。

## 带参数调用

所有 4 个 API 方法都支持传入 `RunnableParams`，用于传递会话标识、用户标识和运行时参数：

```java
RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")      // 会话标识，用于 ChatMemory 和任务管理
        .userId("user_123")              // 用户标识，用于记忆维度
        .build();

// 同步
String answer = agent.call("帮我分析这段代码", params);

// 流式
agent.stream("帮我分析这段代码", params)
        .doOnNext(chunk -> System.out.print(chunk))
        .blockLast();
```

> 参数的详细用法请参考 [03-动态会话参数](03-动态会话参数.md)。

## 内部实现

### 执行引擎

所有 API 最终由 `AgentLoopExecutor` 驱动，核心是 ReAct 循环：

```
用户问题 → [构建消息列表] → [LLM 推理] → [判断是否有工具调用]
                                                    ↓ 是
                                              [执行工具] → [LLM 推理] → ...
                                                    ↓ 否
                                              [返回最终答案]
```

**同步路径**：`doRunLoop()` 方法，在当前线程中同步循环直到完成。

**流式路径**：`scheduleRound()` 方法，基于 Reactor `Sinks.Many<AgentStreamEvent>` 异步推送事件。

### 关键配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `maxRounds` | 100 | 最大执行轮数，防止无限循环 |
| `maxRetries` | 3 | LLM 调用失败时的重试次数 |
| `thinkTagEnabled` | false | 是否启用 `<think/>` 标签解析 |
