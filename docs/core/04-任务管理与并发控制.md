# 任务管理与并发控制

配置 `AgentTaskManager` 后，同一会话的并发流式请求会被拒绝，且支持外部中断。适用于 Web 应用、聊天服务等多用户并发场景。

## 为什么需要任务管理

- **防止并发冲突**：同一会话同时发起多个请求会导致会话记忆混乱、任务状态不一致
- **支持外部中断**：用户取消操作时，需要中断正在执行的流式任务
- **运行时监控**：了解当前有多少任务在运行、哪些会话正在执行

## 启用任务管理

```java
AgentTaskManager taskManager = new AgentTaskManager();

ReactAgent agent = ReactAgent.builder()
        .chatModel(chatModel)
        .taskManager(taskManager)
        .build();
```

不配置 TaskManager 时，Agent 正常工作但不受并发控制和中断能力保护。

## 使用方式

### 启动任务

任务在调用 `stream` / `streamForResult` 时自动注册，需要传入包含 `conversationId` 的 RunnableParams：

```java
RunnableParams params = RunnableParams.builder()
        .conversationId("conv_001")
        .build();

agent.stream("详细介绍一下四大名著", params).subscribe();
```

### 查询任务状态

```java
agent.hasRunningTask("conv_001");          // 查询指定会话是否有运行中任务
agent.getRunningTaskCount();               // 当前运行中的总任务数
```

### 取消任务

```java
agent.stopStream("conv_001");              // 中断指定会话的流式任务
```

取消后，对应的 Flux 流会正常完成（`Complete` 事件），不会产生异常。

### 并发拒绝

如果同一会话已有运行中的任务，再次发起请求会收到 `CONCURRENT_EXECUTION` 错误：

```java
// 第一次请求
agent.stream("问题1", params).subscribe();

// 第二次请求（同一 conversationId）
// 同步：抛出 AgentException(CONCURRENT_EXECUTION)
String answer = agent.call("问题2", params);

// 高级 API：返回 AgentResult.Failed
AgentResult result = agent.callForResult("问题2", params);
// result instanceof AgentResult.Failed f -> f.code() == CONCURRENT_EXECUTION

// 流式：发出 Error 事件后 Complete
agent.streamForResult("问题2", params)
        .doOnNext(event -> {
            if (event instanceof AgentStreamEvent.Error e) {
                // e.code() == CONCURRENT_EXECUTION
            }
        })
        .blockLast();
```

## 典型场景

### Web 聊天服务

```java
@RestController
@RequestMapping("/api/chat")
public class ChatController {

    private final ReactAgent agent;

    @PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<AgentStreamEvent> chat(@RequestBody ChatRequest request) {
        RunnableParams params = RunnableParams.builder()
                .conversationId(request.getConversationId())
                .userId(request.getUserId())
                .build();

        return agent.streamForResult(request.getMessage(), params);
    }

    @PostMapping("/stop")
    public void stop(@RequestParam String conversationId) {
        agent.stopStream(conversationId);
    }

    @GetMapping("/status")
    public Map<String, Object> status(@RequestParam String conversationId) {
        return Map.of(
            "running", agent.hasRunningTask(conversationId),
            "totalTasks", agent.getRunningTaskCount()
        );
    }
}
```

### 用户取消操作

```java
// 前端用户点击"停止生成"按钮
// 后端收到取消请求
agent.stopStream(conversationId);

// 正在执行的 Flux 流会正常结束，不会抛异常
// 前端收到 Complete 事件后关闭连接
```
